# KnowledgeDB App — 요구사항 정의서 (v1.0 확정)

> 상태: **확정** (2026-07-17). 결정 근거는 `03_결정필요사항_체크리스트.md` 참조.

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 목적 | PC의 `Knowledge_DB` 폴더에 축적 중인 HTML 지식 문서를 핸드폰에서 쾌적하게 열람 |
| 사용자 | 본인 (개인용) → 추후 지인에게 URL 공유로 배포 가능성 있음 |
| 대상 플랫폼 | **Android** (기준 기기: Galaxy S25). 태블릿 별도 대응 없음 (폰과 동일 화면) |
| 기술 방식 | **PWA** — GitHub Pages 호스팅 + 홈 화면 추가. 추후 필요 시 TWA/Capacitor로 승격 |
| 콘텐츠 배달 | PC에서 `git push` → GitHub Pages 자동 반영 |
| 콘텐츠 | 학습·정리용 인터랙티브 HTML 문서 (현재 4개, 지속 추가 예정) |

## 2. 콘텐츠(원본 자산) 분석

앱 설계의 전제가 되는 원본 HTML의 특성. **요구사항 도출 근거**이므로 변경 시 갱신할 것.

| 특성 | 현황 | 앱에 미치는 영향 |
|---|---|---|
| 파일 구성 | HTML 4개 (54~118KB), CSS/JS 인라인 | 문서 자체는 단일 파일로 이동 가능 |
| 이미지 | `Docs/img/*.svg` **상대경로** 참조 (32개) | HTML만 복사하면 그림 깨짐 → 폴더 구조 유지 필수 |
| 폰트 | Pretendard·JetBrains Mono를 **CDN에서 로드** | 오프라인 시 폰트 깨짐 → 폰트 로컬 번들 필요 |
| 반응형 | 3개는 `@media` 있음(사이드바 숨김/햄버거), `ai-hw-processors.html`은 **모바일 대응 없음** | Phase 1에서 반응형 보완 필요 |
| 문서 내 탐색 | 각 문서에 자체 사이드바 TOC 존재 | 앱이 문서 내부 TOC까지 만들 필요는 없음 |
| 목록/인덱스 | 문서 목록 페이지 없음 | 앱 홈(문서 목록) 화면을 새로 만들어야 함 |

## 3. 기능 요구사항 (Functional Requirements)

우선순위: **M**(Must, MVP 필수) / **S**(Should, 2차) / **C**(Could, 여유 시)

| ID | 우선순위 | 요구사항 |
|---|---|---|
| FR-01 | M | 문서 목록 화면: 전체 문서를 제목·주제별로 보여주는 홈 화면 |
| FR-02 | M | 문서 뷰어: HTML 문서를 원본 스타일(디자인 시스템) 그대로 렌더링 |
| FR-03 | M | 문서 내 인터랙션(스크롤 스파이, 햄버거 메뉴, JS 동작) 정상 작동 |
| FR-04 | M | SVG 이미지 정상 표시 (상대경로 유지) |
| FR-05 | M | 새 문서 추가: `git push` → Pages 반영 → 홈 화면 목록 자동 갱신 |
| FR-06 | S | **문서별 다운로드 버튼**: 받아둔 문서는 오프라인(비행기 모드)에서도 열람 가능. 이미지·폰트 포함 캐싱 |
| FR-07 | S | 최근 본 문서 / 읽던 위치 기억 |
| FR-08 | S | 문서 제목 검색 |
| FR-09 | S | 다크 모드: 기본은 시스템 설정을 따르고, 사용자가 수동으로 켜고 끌 수 있는 토글 제공 |
| FR-10 | C | 문서 본문 전체 검색 |
| FR-11 | C | 즐겨찾기 / 북마크 (문서 내 특정 섹션) |
| FR-12 | C | 문서별 태그/카테고리 분류 (문서 수가 늘어나면) |

> D6 답변에 따라 FR-06~12 전부 최종 범위에 포함. 순서만 M→S→C로 단계화.

## 4. 비기능 요구사항 (Non-Functional Requirements)

| ID | 요구사항 |
|---|---|
| NFR-01 | 모바일 화면(360~430px 폭)에서 가로 스크롤 없이 읽을 수 있어야 함 |
| NFR-02 | 폰트는 사이트에 번들하여 오프라인·CDN 장애 시에도 동일하게 표시 |
| NFR-03 | 문서 열기 체감 지연 1초 이내 (현재 파일 크기면 문제없음) |
| NFR-04 | 콘텐츠 추가 워크플로는 "git push 한 번"으로 완결 |
| NFR-05 | 유지보수 부담 최소화: 별도 서버·빌드 파이프라인 없이 정적 호스팅으로 운영 |
| NFR-06 | 원본(`Knowledge_DB`)이 source of truth — 앱 쪽에서 문서를 편집하지 않음 (읽기 전용) |
| NFR-07 | 저장소는 public (Pages 무료 사용) — 공개 곤란한 문서는 올리지 않는 것을 원칙으로 함 |
| NFR-08 | Chrome 및 Samsung Internet(갤럭시 기본 브라우저)에서 설치·동작 확인 |

## 5. 범위 제외 (Out of Scope)

- 앱 내 문서 작성/편집 기능
- 다중 사용자, 로그인, 서버 DB
- 수익화, 광고
- iOS 대응 (PWA 특성상 대부분 동작은 하겠지만, 테스트·보증 범위 아님)
