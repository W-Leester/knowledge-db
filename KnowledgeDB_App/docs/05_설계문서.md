# KnowledgeDB App — 설계문서 (v1.0)

> 2026-07-17 작성. 요구사항정의서 v1.0 기준.
> 기술 방식: PWA (GitHub Pages 정적 호스팅), 프레임워크·빌드 도구 없는 순수 HTML/CSS/JS.

## 1. 설계 원칙

1. **빌드 없음**: 푸시한 파일이 곧 서비스다. 번들러·프레임워크·서버 코드 없음 (NFR-05).
2. **문서는 손대지 않는 게 기본**: 원본 HTML은 자립적 문서로 유지. 앱 공통 기능(테마·위치기억)은
   각 문서에 `<script>` 태그 1줄 추가로 주입한다 — 문서 재작성 없음.
3. **`docs.json`이 단일 진실 공급원**: 문서 목록·검색·다운로드 자산·버전을 전부 이 파일 하나로 관리.
   새 문서 추가 = HTML 파일 + docs.json 항목 1개 + push.
4. **점진적 강화**: Service Worker나 JS가 실패해도 문서 열람(핵심 기능)은 항상 동작해야 한다.

```
        │
```

## 3. 저장소·배포 구조

NFR-04 "push 한 번" 원칙이 깨지고, 원본과 배포본 이원화로 관리 부담만 는다.)

```

├── index.html               ← 홈 화면 (완성, PWA 요소는 추가 필요)
├── database-systems.html    ← 문서들 (반응형 완료)
├── ai-hw-processors.html

├── Docs/img/*.svg           ← 문서 이미지 (상대경로 유지)
├── docs.json                ← [신규] 문서 메타데이터 매니페스트
├── manifest.webmanifest     ← [신규] PWA 매니페스트
├── sw.js                    ← [신규] Service Worker (루트에 있어야 전체 스코프)
├── assets/
│   ├── fonts/               ← [신규] Pretendard·JetBrains Mono woff2 (CDN 대체)

│   ├── app.js               ← [신규] 문서 페이지 주입용 공통 스크립트 (테마·위치기억·홈버튼)
```

- `Knowledge_DB.zip`은 .gitignore로 제외 (저장소에 불필요).

## 4. 핵심 컴포넌트 설계

### 4.1 docs.json — 문서 매니페스트

```json
{
  "docs": [
    {
      "id": "database-systems",
      "title": "데이터베이스 완전 정리",
      "desc": "SQL·NoSQL·NewSQL부터 스토리지(S3·CDN)와 Supabase까지",
      "file": "database-systems.html",
      "group": "Software · Data",
      "icon": "🗄",
      "gradient": ["#002C5F", "#1A5CB5"],
      "tags": ["RDBMS", "NoSQL", "S3·CDN", "Supabase"],
      "version": 1,
      "assets": ["Docs/img/db-history-timeline.svg", "…해당 문서가 참조하는 svg 전부"]
    }
  ]
}
```

- **용도**: ① 홈 화면 목록 렌더링 ② 제목/태그 검색 ③ 다운로드 시 캐싱할 파일 목록 ④ 버전 비교로 갱신 감지.
- `version`은 문서 내용을 수정할 때 수동으로 +1 (다운로드해둔 폰에 "새 버전 있음" 표시하는 근거).
- index.html의 하드코딩된 카드 목록은 docs.json 기반 JS 렌더링으로 교체한다.
  (JS 실패 시를 대비해 `<noscript>` 정적 링크 제공 — 원칙 4)

### 4.2 Service Worker (sw.js) — 캐싱 전략

| 대상 | 전략 | 이유 |
|---|---|---|
| 앱 셸: index.html, docs.json, assets/*, icons/* | **사전 캐싱** (SW 설치 시) + stale-while-revalidate | 홈 화면은 오프라인에도 항상 뜨고, 온라인이면 뒤에서 갱신 |
| 문서 HTML + SVG (다운로드 안 한 것) | **network-first**, 캐시 폴백 | 온라인이면 항상 최신, 오프라인이면 이전에 본 사본이라도 표시 |
| 다운로드한 문서 | **cache-first** (`kdb-docs` 캐시) | 사용자가 명시적으로 받아둔 것 — 즉시·오프라인 열람 보장 |

- 캐시 이름: `kdb-shell-v{n}` / `kdb-docs` (문서 캐시는 SW 업데이트에도 유지).
- SW 갱신: sw.js 내 `SHELL_VERSION` 상수 변경 시 구버전 셸 캐시 삭제.

### 4.3 다운로드 버튼 (FR-06)

- 홈 화면 각 문서 카드에 ⬇ 버튼. 탭하면:
  1. docs.json의 해당 문서 `file` + `assets` + 폰트를 `caches.open('kdb-docs')`에 `addAll`
  2. 완료 시 버튼이 ✓(저장됨) 상태로 전환, "오프라인 가능" 배지 표시
  3. 저장 여부는 Cache Storage 존재 여부로 판정 (별도 DB 불필요)
- 갱신: 온라인 상태에서 docs.json의 `version` > 저장 시점 버전이면 카드에 "업데이트" 배지 → 탭하면 재다운로드.
- 삭제: 저장됨 상태 길게 누르기(또는 ⓧ) → 해당 문서 캐시 항목 삭제.
- 오프라인에서 미다운로드 문서 탭 → "오프라인입니다. 이 문서는 다운로드되지 않았어요" 안내 화면 (SW 폴백).

### 4.4 다크 모드 (FR-09)

- 모든 문서가 동일한 디자인 토큰(`--bg-primary`, `--navy-*` 등 CSS 변수)을 쓰므로,
  **공통 `dark.css` 하나로 `:root[data-theme="dark"]`에서 변수만 오버라이드**하면 전 문서에 적용된다.
- 상태: `localStorage.kdb-theme` = `"system" | "light" | "dark"` (기본 `system`).
  - `system`일 때 `prefers-color-scheme: dark` 매치 → data-theme="dark" 부여.
- 토글 UI: 홈 화면 헤더에 3단 토글(시스템/라이트/다크). 문서 페이지에서는 app.js가 플로팅 버튼 제공.
- FOUC 방지: `<head>` 최상단 인라인 스니펫에서 localStorage 읽어 data-theme 즉시 설정.

### 4.5 문서 페이지 주입 스크립트 (assets/app.js)

각 문서 `</body>` 앞에 한 줄 추가: `<script src="assets/app.js" defer></script>`
(+ `<head>`에 `dark.css` 링크, 폰트 CDN 링크 → `fonts.css` 교체 — 문서당 3줄 수정이 전부)

기능:
- **읽던 위치 기억** (FR-07): scroll 위치를 `localStorage.kdb-pos:{id}`에 저장(디바운스),
  재진입 시 복원 + "이어서 읽기" 토스트
- **최근 본 문서**: 진입 시 `localStorage.kdb-recent`에 [id, timestamp] 기록 → 홈 화면 "최근" 섹션
- **홈 버튼**: 좌상단 플로팅 ⌂ 버튼 (현재 문서에서 index로 돌아갈 링크가 없음 — Android 뒤로가기도
  되지만 명시적 수단 제공)
- **테마 적용/토글** (4.4)

```json
  "display": "standalone",
  "background_color": "#F2EDE4",

```

아이콘은 디자인 시스템 네이비(#002C5F) 배경에 심볼 — SVG로 제작 후 PNG 변환.

### 4.7 검색 (FR-08 → FR-10)

- **MVP**: 홈 화면 상단 검색바 — docs.json의 title/desc/tags를 클라이언트에서 필터 (수 KB, 즉시).
- **Phase 4 (본문 전체 검색)**: `tools/build-search-index.js` (Node)가 각 HTML의 텍스트를 추출해
  `search-index.json` 생성 (섹션 제목 단위). 문서 추가 시 스크립트 1회 실행 후 push.
  결과 탭 → 해당 문서의 섹션 앵커로 이동. *(전 단계 중 유일하게 "빌드 유사" 단계이나 선택적)*

## 5. 클라이언트 저장소 설계

| 저장소 | 키/이름 | 내용 |
|---|---|---|
| localStorage | `kdb-theme` | `system` \| `light` \| `dark` |
| localStorage | `kdb-pos:{docId}` | 스크롤 위치(px)와 저장 시각 |
| localStorage | `kdb-recent` | 최근 본 문서 [{id, ts}] 최대 10개 |
| localStorage | `kdb-dl-ver:{docId}` | 다운로드 시점의 문서 version |
| Cache Storage | `kdb-shell-v{n}` | 앱 셸 (홈·폰트·아이콘·docs.json) |
| Cache Storage | `kdb-docs` | 다운로드한 문서 HTML + SVG |

## 6. 운영 워크플로 (문서 추가/수정)

```
새 문서 추가:
  1. Knowledge_DB/에 새 HTML 작성 (디자인 시스템 + 반응형 + app.js/fonts.css/dark.css 3줄 포함)
  2. docs.json에 항목 추가 (assets 배열에 참조 SVG 나열)
  3. git push  →  폰에서 홈 화면 새로고침 시 자동 등장

기존 문서 수정:
  1. HTML 수정  2. docs.json의 version +1  3. git push
  → 다운로드해둔 폰에는 "업데이트" 배지
```

## 7. 화면 설계

### 홈 (index.html)
```
┌──────────────────────────────┐
│ 헤더 (네이비 그라데이션)      │  ← 타이틀 + 테마 토글(◐)
│ [ 🔍 검색바 ]                 │
├──────────────────────────────┤
│ 최근 본 문서 (가로 스크롤 칩) │  ← localStorage 기반, 없으면 숨김
│ ── Software · Data ──        │
│ [카드: 아이콘|제목|태그  ⬇/✓] │  ← 탭=열기, ⬇=다운로드, ✓=저장됨
│ ── AI Hardware ──            │
│ [카드] [카드]                 │
└──────────────────────────────┘
```

### 문서 페이지 (원본 유지 + 주입 요소)
- 원본 레이아웃 그대로. app.js가 추가하는 것: ⌂ 홈 플로팅 버튼(좌상단), 테마 토글(우상단),
  이어읽기 토스트. 기존 문서 자체 컨트롤(햄버거 등)과 위치 겹치지 않게 배치.

## 8. 엣지 케이스 / 실패 모드

| 상황 | 동작 |
|---|---|
| JS 비활성/로드 실패 | `<noscript>` 정적 문서 링크 목록 → 열람은 가능 (원칙 4) |
| 오프라인 + 미다운로드 문서 | SW가 안내 페이지 폴백 |
| 오프라인 + 홈 첫 방문 아님 | 셸 캐시로 홈 표시, 다운로드 문서만 열람 가능 |
| docs.json과 실제 파일 불일치 | 문서 열기 404 → SW 폴백 페이지. (docs.json 갱신 누락이 유일한 휴먼에러 지점 — README에 체크리스트화) |
| Samsung Internet | PWA 설치·SW 지원됨. Chrome과 함께 NFR-08 테스트 대상 |

## 9. 남은 작업 (로드맵 매핑)

Phase 1 잔여: 폰트 로컬화(fonts.css), dark.css, app.js, docs.json, 문서 4개에 3줄 패치, 아이콘 제작
Phase 2: .gitignore, git 저장소화, GitHub Pages 배포, manifest.webmanifest, S25 설치 테스트
Phase 3: sw.js (셸 캐싱 → 다운로드 버튼 → 갱신 배지), 위치 기억, 제목 검색
Phase 4: 본문 검색 인덱스, 북마크, 태그 필터
