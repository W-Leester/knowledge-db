## D3. 콘텐츠 업데이트 방식 ★

새 HTML 문서를 계속 만들 예정이므로, 폰까지 배달하는 파이프라인이 핵심 결정.

- [ ] 어느 방식이 좋나요?
  - **(a) GitHub Pages 호스팅** — PC에서 `git push`만 하면 폰에서 바로 최신 문서. 단, 저장소가 public이면 누구나 URL로 접근 가능 (URL을 모르면 사실상 못 찾지만, 비밀은 아님)
  - **(b) 앱에 파일 내장** — 완전 오프라인·완전 비공개. 대신 문서 추가할 때마다 앱 재빌드+재설치
  - **(c) 클라우드 드라이브 동기화** (Google Drive 등) — 중간 형태, 구현 복잡도는 더 높음
- [ ] 문서에 외부 공개가 곤란한 내용이 있나요? (있으면 (a) public은 배제)
- [ ] 새 문서를 얼마나 자주 추가할 것 같나요? (주 1회 이상이면 (b)는 번거로움)

**답변:** a
